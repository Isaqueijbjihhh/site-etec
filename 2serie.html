2º ano: Aps – Analise e Projetos de sistemas

Resumo:

No 2º ano do curso de Análise e Desenvolvimento de Sistemas (ADS), também conhecido como Análise e Projetos de Sistemas, o foco começa a se aprofundar em desenvolvimento de software, banco de dados, engenharia de software e outras áreas mais técnicas.:

Matérias do 2º ano de APS

• 1. Estrutura de Dados Listas, filas, pilhas, árvores, grafos. Algoritmos para busca e ordenação. Eficiência de algoritmos (complexidade)..

•2. Engenharia de Software Modelagem e documentação de sistemas. Ciclo de vida de software (ágil, cascata). Requisitos, testes, manutenção. .

•3. Programação Orientada a Objetos (POO) Conceitos como herança, polimorfismo, encapsulamento. Aplicações práticas em linguagens como Java, C# ou Python. .

• 4. Banco de Dados II Stored procedures, triggers, views. Normalização mais avançada. Administração e segurança de dados..

• 5. Desenvolvimento Web (Front-end e/ou Back-end) HTML, CSS, JavaScript. Frameworks como React, Angular, Node.js, PHP, etc. APIs RESTful.

6. Análise e Projeto de Sistemas II Modelagem UML mais avançada (casos de uso, diagramas de sequência, classes). Levantamento e análise de requisitos. Projetos em grupo (muitas vezes simulando cliente real).

7. Segurança da Informação (em algumas grades) Conceitos de criptografia, autenticação. Políticas de segurança e boas práticas.

8. Projeto Integrador / APS (Atividades Práticas Supervisionadas) Desenvolvimento de um sistema completo em grupo. Aplicação prática de tudo o que foi aprendido. Apresentações e relatórios.

Dicas para se dar bem no 2º ano:

Comece a estudar Git/GitHub: será essencial para trabalho em equipe. Pratique programação fora das aulas (em sites como HackerRank, LeetCode, etc.). Use ferramentas de modelagem como StarUML ou Draw.io. Aprenda a usar um banco de dados de verdade, como PostgreSQL, MySQL ou SQL Server. Se quiser, posso te ajudar a montar um plano de estudos baseado na sua grade específica. Você estuda em qual instituição? Posso procurar a grade curricular exata se me disser o nome e campus..

Projetos típicos da disciplina:
Desenvolvimento de protótipos de sistemas.
Soluções para problemas técnicos específicos.
Estudos de caso aplicados a cenários reais.
Apresentações e relatórios técnicos detalhados.

2º ano: PW – Programação Web

Resumo:

A disciplina de PW (Programação Web) tem como objetivo introduzir os alunos ao desenvolvimento de sites e aplicações para a web, desde o front-end (parte visual) até conceitos básicos de back-end.

• HTML: linguagem de marcação usada para estruturar o conteúdo da web (textos, formulários, etc.)
• CSS: linguagem de estilo usada para deixar os sites bonitos e responsivos (cores, fontes.).

• 1. Front-End Intermediário/Avançado HTML5 e CSS3 (reforço e boas práticas). JavaScript (JS): manipulação de DOM, eventos, validação de formulários. Uso de bibliotecas/frameworks: jQuery (em alguns cursos mais tradicionais). Frameworks modernos (React, Vue ou Angular – depende da instituição).

• 2. Back-End com Linguagem Web PHP, Node.js, Python (Django/Flask) ou Java (Spring) – depende da faculdade. Lógica de negócio no servidor. Integração com banco de dados (MySQL, PostgreSQL, etc.). Upload de arquivos, sessões, cookies. .

3. Banco de Dados em Aplicações Web CRUD (Create, Read, Update, Delete). Conexão entre front-end e banco via scripts back-end. Segurança nas queries (uso de SQL preparado / PDO, por exemplo).

4. Criação e Consumo de APIs (REST) O que é uma API. Como criar uma API simples. Como consumir uma API usando JavaScript (fetch, axios). JSON como formato de dados.

5. Projeto Final / Sistema Web Completo Um sistema web funcional com: Front-end interativo. Back-end com regras de negócio. Banco de dados. Exemplo: sistema de cadastro, login de usuários, gerenciamento de produtos etc.

2º ano: DD-Design Digital

Resumo:

• 1. Fundamentos do Design Princípios do design: alinhamento, contraste, repetição, proximidade. Cores: teoria das cores, combinações, acessibilidade. Tipografia: escolha de fontes, legibilidade, hierarquia visual. Layout e grid: organização de elementos na tela.

• 2. UI/UX (Interface e Experiência do Usuário) Diferença entre UI (User Interface) e UX (User Experience). Navegação intuitiva e usabilidade. Personas e jornada do usuário. Design centrado no usuário..

• 3. Ferramentas de Design Ferramentas comuns: Figma (mais usado atualmente, gratuito e online). Adobe XD (em algumas instituições). Canva ou Sketch (menos comum em ADS, mas pode aparecer). Prototipagem de telas: wireframes, mockups e protótipos interativos.

•1. Fundamentos do Design Visual Princípios de design: alinhamento, contraste, repetição, proximidade. Teoria das cores: psicologia das cores, combinações, paletas. Tipografia: tipos de fontes, hierarquia visual, legibilidade. •Layouts e grids: organização visual da interface. .

5. Ferramentas mais usadas
•Figma (principal ferramenta atual, gratuita e online).
•Adobe XD (menos usado, mas ainda presente em algumas faculdades).
•Canva (para identidade visual, em alguns casos).
•Icons8, Google Fonts, Coolors (suporte visual).

Objetivo geral:

Desenvolver a capacidade de criar interfaces bem estruturadas, que proporcionem uma boa experiência ao usuário e valorizem os sistemas desenvolvidos.

2º ano: Bd-Banco de dados

Resumo:

Essa disciplina ensina como projetar, implementar e gerenciar bancos de dados relacionais. Durante as aulas, os alunos aprendem:

• 1. SQL Avançado Subconsultas (subqueries). JOINs complexos (INNER, LEFT, RIGHT, FULL). Funções agregadas (SUM, AVG, COUNT, etc.). Agrupamentos com GROUP BY e filtros com HAVING.

•1. SQL Avançado Subconsultas (subqueries). JOINs complexos (INNER, LEFT, RIGHT, FULL). Funções agregadas (SUM, AVG, COUNT, etc.). Agrupamentos com GROUP BY e filtros com HAVING.

• 3. Triggers (Gatilhos) Automatizar ações no banco após INSERT, UPDATE, DELETE. Exemplo: atualizar um log toda vez que um registro é alterado..

• 4 Views (Visões) Criação de visões com CREATE VIEW. Simplificar consultas complexas. Segurança: restringir colunas visíveis a certos usuários..

• 5. Normalização (Revisão e Aplicação Prática) Formas normais (1FN até 3FN). Eliminar redundância e garantir integridade..

• 6. Transações e Controle de Concorrência BEGIN, COMMIT, ROLLBACK. Propriedades ACID: Atomicidade Consistência Isolamento Durabilidade Controle de acesso simultâneo ao banco (lock, deadlock).

7. Administração e Segurança Criação de usuários e permissões (GRANT, REVOKE). Backups e restore. Monitoramento de desempenho (índices, EXPLAIN, etc.).

Objetivo geral:

Habilitar o aluno a desenvolver sistemas que utilizam bancos de dados para armazenar informações de forma estruturada, segura e eficiente.

2º ano: Fd-Fundamentos da informatica

Resumo:

Objetivo: fornecer a base essencial sobre como os computadores funcionam, seus componentes, sistemas operacionais e lógica básica de funcionamento da tecnologia.:

•1. Conceitos Básicos de Informática O que é hardware e software. Tipos de computadores e sistemas embarcados. Entradas, saídas e dispositivos de armazenamento.

•2. Arquitetura de Computadores CPU (Unidade Central de Processamento). Memória RAM e ROM. HD, SSD, placas-mãe e periféricos. Barramentos e componentes internos.
•3. Sistemas Operacionais Funções básicas de um SO. Windows, Linux e outros exemplos. Gerenciamento de processos, arquivos e usuários.

• 4. Redes de Computadores (noções básicas) O que é uma rede e como ela funciona. Tipos de redes (LAN, WAN, MAN). Internet, IP, DNS, protocolos (TCP/IP).
• 5. Pacote Office / Ferramentas Digitais Word, Excel, PowerPoint (uso básico). Google Drive e ferramentas na nuvem. Softwares de produtividade.
• 6. Segurança da Informação (noções básicas) Vírus, malwares, phishing. Antivírus e boas práticas de segurança. Senhas seguras, backups.
• 7. História da Computação Evolução dos computadores. Principais inventores e marcos históricos. Importância da informática na sociedade.

Noções básicas de responsividade, design visual, estruturas de navegação, e boas práticas no desenvolvimento web.

Objetivo geral:

Capacitar o aluno a desenvolver soluções computacionais básicas e intermediárias, utilizando lógica estruturada e boas práticas de programação.

2º ano: Tpa-Técnicas de Programação e Algoritmos

Resumo:

aprofundar a lógica de programação, trabalhar com estruturas de dados simples, modularização, e começar a resolver problemas mais complexos com algoritmos otimizados.

• 1. Revisão Rápida (nível avançado) Sintaxe da linguagem (Python, C, Java, depende da instituição). Estruturas condicionais e de repetição. Funções com retorno, passagem por referência e valor.

•2. Vetores e Matrizes Avançadas Manipulação complexa (ordenação, filtros, buscas). Algoritmos de ordenação: Bubble Sort, Selection Sort, Insertion Sort. Busca linear e binária.

•3. Strings e Manipulação de Texto Leitura e análise de textos. Contagem de vogais, palavras, ou formatação de strings. Conversão de dados (string → número e vice-versa).
•4. Subprogramas (Funções e Procedimentos) Modularização do código. Uso de múltiplas funções para dividir problemas. Organização por arquivos (em linguagens como C e Python).

• 5. Recursividade (caso apareça na grade) Funções que chamam a si mesmas. Fatorial, Fibonacci, Torres de Hanoi. Comparação entre recursão e laços.
•6. Algoritmos de Lógica Complexa Verificação de CPF ou CNPJ (exercícios reais). Algoritmos de criptografia simples (ex: Cifra de César). Problemas de programação competitiva (nível iniciante).


Objetivo geral:

Capacitar o aluno a desenvolver soluções computacionais básicas e intermediárias, utilizando lógica estruturada e boas práticas de programação.
